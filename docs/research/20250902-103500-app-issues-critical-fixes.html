  • CameraGalleryActivity.kt:164-176 - Gallery photo loading

üìπ Issue 2: Camera Viewfinder Alignment

Problem Description

The viewfinder overlay and aspect ratio calculations produce incorrect dimensions in portrait mode. The 3:4 and 9:16 ratios appear too short, and the overlay doesn't align with the actual capture area.

Root Cause Analysis

The aspect ratio calculation logic in ViewfinderOverlay.kt:96-104 assumes landscape orientation:

// Current logic (problematic in portrait): val (windowWidth, windowHeight) = if (canvasWidth / canvasHeight > targetRatio) { val width = canvasHeight * targetRatio Pair(width, canvasHeight) } else { val height = canvasWidth / targetRatio // ‚Üê Problem here Pair(canvasWidth, height) }

Mathematical Analysis

In portrait mode with screen ratio 9:16 (0.56):

  • 4:3 ratio (1.33): Goes to else branch ‚Üí height too short
  • 16:9 ratio (1.77): Goes to else branch ‚Üí height much too short

üí° Recommended Solution

Orientation-Aware Calculations:

  1. Detect device orientation (portrait/landscape)
  2. Adjust target ratio based on orientation
  3. Use reciprocal ratios for portrait mode (3:4 instead of 4:3)
  4. Ensure overlay bounds match camera preview bounds

Key Files

  • CameraScreen.kt:86-90 - Aspect ratio definitions
  • ViewfinderOverlay.kt:96-104 - Critical calculation logic
  • ViewfinderOverlay.kt:40-143 - Overlay system

‚öôÔ∏è Issue 3: Settings Drawer UI Instability

Problem Description

When the settings drawer opens, the gallery, capture, and settings icons move unexpectedly. This creates visual instability and reduces user confidence in the interface.

Root Cause Analysis

The settings drawer uses fixed offset positioning instead of proper layout constraints:

// Problematic fixed positioning: AnimatedVisibility( visible = showQuickActions, modifier = Modifier .align(Alignment.CenterEnd) .offset(x = (-8).dp, y = (-80).dp) // ‚Üê Fixed offset )

Layout Architecture Issues

  • Settings button uses align(Alignment.CenterEnd)
  • Quick actions drawer uses fixed offset()
  • No space reservation for drawer, causing layout reflow
  • Animation conflicts with parent layout measurements

üí° Recommended Solution

Constraint-Based Layout:

  1. Replace fixed offset with proper layout constraints
  2. Use Scaffold with drawer or dedicated drawer component
  3. Reserve space for drawer to prevent content reflow
  4. Ensure animation doesn't affect other UI elements

Key Files

  • CameraScreen.kt:1048-1136 - Quick actions drawer
  • CameraScreen.kt:1034-1046 - Settings button positioning
  • CameraScreen.kt:978-1031 - Main controls layout
